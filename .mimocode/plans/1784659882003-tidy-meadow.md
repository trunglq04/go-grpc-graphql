# Fix Elasticsearch functions to use TypedClient properly

## File to modify
- `catalog/repository.go` (lines 91-186)

## Problem
Three functions have issues preventing compilation or using wrong API patterns with `go-elasticsearch/v9` `TypedClient`.

## Changes

### 1. `ListProducts` (lines 91-114)

**Issues:**
- Uses `res.Body` / `json.NewDecoder` to decode response — TypedClient's `Search().Do()` returns a typed `*search.Response`, not a raw HTTP response
- References undefined variable `esResponse`

**Fix:** Remove the body decoding. Iterate `res.Hits.Hits` directly, unmarshal each `hit.Source_` into `productDocument`. Use `*hit.Id_` for the ID.

```go
func (r *elasticRepository) ListProducts(ctx context.Context, skip, take uint64) ([]Product, error) {
	res, err := r.client.Search().Index(IndexName).From(int(skip)).Size(int(take)).Do(ctx)
	if err != nil {
		return nil, err
	}
	if len(res.Hits.Hits) == 0 {
		return nil, ErrNotFound
	}

	products := make([]Product, 0, len(res.Hits.Hits))
	for _, hit := range res.Hits.Hits {
		var source productDocument
		if err := json.Unmarshal(hit.Source_, &source); err != nil {
			return nil, err
		}
		products = append(products, Product{
			ID:          *hit.Id_,
			Name:        source.Name,
			Description: source.Description,
			Price:       source.Price,
		})
	}
	return products, nil
}
```

### 2. `ListProductsWithIDs` (lines 116-145)

**Issues:**
- `res.Docs` is `[]types.MgetResponseItem` (which is `any`). Type assertion `item.(*types.GetResult)` uses pointer assertion — the JSON decoder may deserialize as value type. Use a type switch for robustness.

**Fix:** Replace type assertion with type switch handling both `types.GetResult` and `*types.GetResult`.

```go
func (r *elasticRepository) ListProductsWithIDs(ctx context.Context, ids []string) ([]Product, error) {
	res, err := r.client.Mget().
		Index(IndexName).
		Ids(ids...).
		Do(ctx)
	if err != nil {
		return nil, err
	}

	products := make([]Product, 0, len(res.Docs))
	for _, item := range res.Docs {
		switch doc := item.(type) {
		case types.GetResult:
			if !doc.Found {
				continue
			}
			var source productDocument
			if err := json.Unmarshal(doc.Source_, &source); err != nil {
				return nil, err
			}
			products = append(products, Product{
				ID:          doc.Id_,
				Name:        source.Name,
				Description: source.Description,
				Price:       source.Price,
			})
		case *types.GetResult:
			if !doc.Found {
				continue
			}
			var source productDocument
			if err := json.Unmarshal(doc.Source_, &source); err != nil {
				return nil, err
			}
			products = append(products, Product{
				ID:          doc.Id_,
				Name:        source.Name,
				Description: source.Description,
				Price:       source.Price,
			})
		}
	}
	return products, nil
}
```

### 3. `searchProducts` (lines 147-186)

**Issues:**
- Uses `r.client.Search(ctx, r.client.Search.WithContext(ctx), ...)` — this is the low-level `Client.Search()` method, NOT the `TypedClient` fluent API
- Uses `res.Body`, `res.IsError()`, `json.NewDecoder(res.Body)` — all low-level patterns
- TypedClient's `Search()` returns a `*core_search.Search` builder, not the response directly

**Fix:** Convert to TypedClient fluent API. Use `types.Query{QueryString: &types.QueryStringQuery{Query: query}}` to build the query DSL. Iterate `res.Hits.Hits` for results.

```go
func (r *elasticRepository) searchProducts(ctx context.Context, query string, skip uint64, take uint64) ([]Product, error) {
	res, err := r.client.Search().
		Index(IndexName).
		Query(&types.Query{QueryString: &types.QueryStringQuery{Query: query}}).
		From(int(skip)).
		Size(int(take)).
		Do(ctx)
	if err != nil {
		return nil, err
	}

	products := make([]Product, 0, len(res.Hits.Hits))
	for _, hit := range res.Hits.Hits {
		var source productDocument
		if err := json.Unmarshal(hit.Source_, &source); err != nil {
			return nil, err
		}
		products = append(products, Product{
			ID:          *hit.Id_,
			Name:        source.Name,
			Description: source.Description,
			Price:       source.Price,
		})
	}
	return products, nil
}
```

Remove `"fmt"` from imports (no longer needed after `searchProducts` fix).
